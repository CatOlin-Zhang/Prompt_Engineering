📝 项目要求与实施指南
1. 模型与 API 限制
运行环境：使用 Ollama 运行本地模型，参数规模上限为 120亿 (12B)。
推荐模型：选择小型开源大语言模型，例如：
gemma3:4b
gemma3:12b
qwen3:9b
核心流程：仅允许使用原始生成方式。
Python：使用 ollama.generate 函数。
HTTP：使用 POST /api/generate 接口。
注：示例用法请参考模板笔记本。
2. 可复现性要求
认证标准：项目需符合 FSC801CD 认证标准。
扣分项：若缺失必要依赖项或导致实验设置不可复现，将进行评分扣分。
3. 令牌效率优化
目标：运用课程中的提示工程方法，在保证回答质量的前提下减少冗余令牌消耗。
评估：效率表现是评分评估的重要指标。
4. 最低要求实施方案
必须至少在一个 Jupyter Notebook 中实现可运行的端到端工作基线（实验课时）。
提示词组装（主题 6）：
需包含清晰指令、上下文信息及应答格式控制。
示例：角色/任务框架设定、明确约束条件、最终答案过渡清晰度。
生成控制（主题 2）：
需设置关键参数（如温度参数与输出长度）。
需在报告中简要说明参数配置依据。
对话管理（主题 3）：
适用于场景下的历史记录处理和/或系统消息控制。
上下文工程化流程（主题 4 & 5）：
生成前需完成检索、片段化/分块处理及相关性优先级排序。
RAG 证据（主题 5）：
需包含至少一种 词汇检索 设置。
需包含至少一种 基于嵌入式检索 设置。
需附带简要的对比分析。
基础评估（主题 4）：
针对代表性查询进行评估。
需包含至少一项 质量对比。
需包含至少一项 词元使用对比。
5. 高级实施方案（可选）
注意：高级方案仅在完成“最低要求实施方案”且技术实现可靠时方可获得额外加分。此部分旨在提供更高的区分度。
课程进阶技术（主题 7-11）：
推理导向提示。
ReAct / 工具增强推理。
对话工作流 / 智能体。
高级评估框架。
安全 / 驯化策略。
教学大纲外技术：
网页 / 移动端 UI 设计。
部署方案。
外部编排或数据库驱动产品功能。
